RESTful API and HTTP Automated Tests

Intro

  • A RESTful API uses HTTP requests to help a client (web, mobile app or client.py) to interact with server's database.
  • HTTP requests (GET, POST or DELETE) allow for retrieving, inserting or deleting data.
  • With RESTful API in Flask, I can define the way of interaction between client and server.
  • Here is an example of a client sending POST request with data to server through API:
  • The representation of a database to API is Python Data Model.
  • Python Data Model is a class that with its attributes reflects data schema.
  • In the project, I define two classes: TransactionModel and PayerModel.
  • The representation of Python Data Model to client App is a defined within API Resource.
  • For each specific Data Model we need to define separate API Resource:
    - for TransactionModel there is need to define Transaction API Resource.
    - for PayerModel there is need to define Payer API Resource.
  • Resource is an external representation of internal data Model.
  • Client App can access Python object of given Model only thorugh API Resource.
  • Client App can interact with Resource using HTTP requests.
  • When client App sends GET request, API accepts it, connects database, formulates response and sends it back to client with JSON
  • When client APP sends POST request, packs all information into JSON, sends to API, API accepts it, parses JSON into python variables which feed python model class
  • With that solution, API Resource deciedes what Model objects are accessible and in what way id est getting, posting, putting, deleting.

Demo

Client - API - Server model:

  1. client hits the api endpoint like server.com/transaction/transaction_id
    - transaction/ in api endpoint redirects to a specific web server spot,
    - transaction_id in api endpoint serves as variable which is passed to api resource.
  2. with hitting endpoint client sends post request passing JSOM parameters along with transaction_id variable
  3. flask server redirects request to API Resource accoring to hit endpoint
  4. request calls post method in API Resource which parses parameters from JSON into Python variables
  5. resource's post method creates Model instance passing obtained varialbes as arguments to class constructor
  6. resource uses Model's methods to save instance into database
